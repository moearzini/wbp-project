Motivation: Dynamische Webanwendungen

Web-Programmierung · Vorlesung 1 · Einstieg über Deployment

Die Motivation wurde über eine einfache Frage aufgebaut: Wie verteile/veröffentliche ich mein fertiges Programm (Deployment)?

Deployment: Installation vs. Web

1) Klassisch: Installation (CD/Download)
  • Benutzer muss installieren & aktualisieren
  • Plattformabhängigkeiten (Windows/Mac/Linux)
  • Support-Aufwand steigt (Versionen, Fehler, Setup)
2) Als Web-Anwendung
  • Keine Installation: Browser reicht
  • Updates zentral auf dem Server
  • Geräteunabhängig (solange Browser vorhanden)

Warum „dynamisch“?

  • Die Seite reagiert auf Eingaben (z. B. Plausibilitätschecks, Dialoge)
  • Inhalte können im Browser geändert werden (über DOM + JavaScript)
  • Daten können im Hintergrund nachgeladen werden (asynchron, später: AJAX)
Merke:
Die zentrale Frage dahinter ist immer auch: Wie sieht die Software-Architektur aus?
Was läuft im Browser (Client) – und was läuft auf dem Server?
